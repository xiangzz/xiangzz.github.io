异常机制概述

程序运行中出现未考虑到的情况时，会抛出异常或错误，影响正常执行。

public static void main(String[] args) {
    test(1, 0);   // 当 b 为 0 时会怎样？
}
private static int test(int a, int b){
    return a / b; // 未进行判断，运行期可能抛出异常
}

核心知识点：

  • 异常本质是类的对象，支持在运行时抛出
  • 分类：运行时异常(RuntimeException)、编译时异常(Exception)、错误(Error)
  • 运行时异常可不强制处理；编译时异常必须捕获或声明
  • 抛出异常使用 throw；方法声明可能抛出异常用 throws
  • 处理异常使用 try、catch、finally，支持多重捕获与顺序要求
  • 断言表达式 assert 在开启断言时用于校验，失败抛 AssertionError
