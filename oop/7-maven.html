为什么需要 Maven？常见开发痛点

  • 找包麻烦：手动下载 JAR 并维护 classpath，效率低且易错。
  • 依赖迷宫：传递性依赖管理困难，易出现 ClassNotFoundException。
  • 打包难题：构建与打包步骤繁琐，难以保证一致性与可复现。

用 Maven 的声明式依赖与标准化构建生命周期，系统性解决以上问题。

什么是 Maven

Maven 是 Apache 基金会开源的项目管理与构建自动化工具，核心理念基于 POM（Project Object Model）。

  • 依赖管理：在 pom.xml 中声明坐标，自动下载并解析传递依赖。
  • 构建生命周期：标准化 clean、compile、test、package、deploy。

示例命令：mvn package 完成编译到打包的全流程。

Maven 的仓库体系

仓库（Repository）是依赖（JAR/插件）存放与分发的核心：本地仓库、中央仓库与镜像仓库。

本地仓库（Local Repository）

  • 位置：默认 ${user.home}\.m2\repository（如 C:\\Users\\你的用户名\\.m2\\repository）。
  • 作用：缓存依赖以加速构建；可在离线情况下复用已下载依赖。
  • 流程：优先从本地查找，缺失时再访问远程仓库。

可通过 settings.xml 自定义本地仓库路径。

中央仓库（Central Repository）

  • 定义：由社区官方维护的全球性远程仓库，收录丰富且权威。
  • 默认依赖来源：本地缺失时自动从中央仓库下载。
  • 访问地址：https://repo.maven.apache.org/maven2/

镜像仓库（Mirror Repository）

  • 目的：加速国内访问、提升可用性。
  • 机制：对被镜像仓库（如 central）的请求自动重定向到镜像地址。

在 settings.xml 中配置镜像：

<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>central</mirrorOf>
    <name>Aliyun Maven</name>
    <url>https://maven.aliyun.com/repository/central</url>
  </mirror>
</mirrors>

在 IDEA 中配置 Maven

IDEA 自带 Maven 环境，可直接在设置中查看与定制。

修改 settings.xml：本地仓库与镜像

自定义本地仓库路径（示例）：

<localRepository>D:\Java\MavenRepository</localRepository>

配置镜像（示例）：

<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>central</mirrorOf>
    <name>Aliyun Maven</name>
    <url>https://maven.aliyun.com/repository/central</url>
  </mirror>
</mirrors>

在 IDEA 中保存并应用以上设置。

创建 Maven 项目

使用 maven-archetype-quickstart 模板快速创建 “纯 Java” 项目（包含 pom.xml、src/main/java、src/test/java）。

Maven 项目结构

遵循 “约定优于配置”：标准目录与构建流程。

POM（项目对象模型）基础示例

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>org.example</groupId>
    <artifactId>MavenTest</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>8</maven.compiler.source>
        <maven.compiler.target>8</maven.compiler.target>
    </properties>
</project>

项目唯一标识：GAV 与属性

  • <groupId>：组织/公司 ID（反向域名，如 org.apache）。
  • <artifactId>：项目/模块 ID（在组内唯一）。
  • <version>：版本（-SNAPSHOT 表示开发版）。

<properties> 用于统一配置（如 Java 版本），避免硬编码。

声明式依赖管理

在 pom.xml 的 <dependencies> 标签内声明所有依赖：

<dependencies>
    <!-- 在这里放置项目所需的所有依赖 -->
</dependencies>

使用 MVNRepository 搜索库坐标：https://mvnrepository.com/

示例：添加 Lombok 依赖

<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.22</version>
    <scope>provided</scope>
</dependency>

保存后，IDEA 会自动下载依赖并在代码中生效。

代码示例：Student 与 Main

import lombok.AllArgsConstructor;
import lombok.Data;

@Data
@AllArgsConstructor
public class Student {
    String name;
    int age;
}
public class Main {
    public static void main(String[] args) {
        Student student = new Student("小明", 18);
        System.out.println(student);
    }
}

输出示例：Student(name=小明, age=18)，说明 Lombok 已生效。
