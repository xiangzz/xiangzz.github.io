基本类型包装类

面向对象程序设计（Java）

掌握包装类的使用，让基本类型支持面向对象操作

包装类介绍

Java并不是纯面向对象的语言，基本数据类型不是面向对象的。

Java提供了基本类型包装类，使得基本类型能够支持对象操作！

基本类型与包装类对应关系：

  • byte → Byte
  • boolean → Boolean
  • short → Short
  • char → Character
  • int → Integer
  • long → Long
  • float → Float
  • double → Double

包装类基本使用

以Integer类为例：

public static void main(String[] args) {
    Integer i = new Integer(10);    // 将10包装为Integer类型
}

包装类实际上就是将基本数据类型封装成一个类：

// Integer类内部结构
private final int value;  // 存储包装的值

public Integer(int value) {
    this.value = value;
}

自动装箱和拆箱

自动装箱：

Integer i = 10;    // 自动装箱
// 等价于：
Integer i = Integer.valueOf(10);

自动拆箱：

Integer i = 10;
int a = i;    // 自动拆箱
// 等价于：
int a = i.intValue();

参与运算：

Integer a = 10, b = 20;
int c = a * b;    // 自动拆箱参与计算
System.out.println(c);  // 输出：200

包装类比较

new创建的对象：

Integer a = new Integer(10);
Integer b = new Integer(10);
System.out.println(a == b);    // false，不是同一个对象

自动装箱的对象：

Integer a = 10, b = 10;
System.out.println(a == b);    // true，在缓存范围内

IntegerCache机制：

  • 默认缓存 -128~127 之间的值
  • 在此范围内的自动装箱会返回同一个对象
  • 超出范围则创建新对象
Integer a = 128, b = 128;
System.out.println(a == b);    // false，超出缓存范围

包装类常用方法

字符串转换：

Integer i = new Integer("666");   // 字符串转Integer
Integer j = Integer.valueOf("5555");
Integer k = Integer.parseInt("5555");

进制转换：

// 十六进制字符串转整数
Integer i = Integer.decode("0xA6");
System.out.println(i);  // 输出：166

// 整数转十六进制字符串
System.out.println(Integer.toHexString(166));  // 输出：a6

大数处理类 - BigInteger

重要说明：BigInteger不是包装类，它没有对应的基本数据类型！

特点：

  • 用于表示任意精度的整数，大小理论上只受内存限制
  • 不支持自动装箱/拆箱（因为没有对应的基本类型）
  • 必须使用方法进行运算，不能使用 +、-、*、/ 运算符
  • 不可变对象，每次运算都产生新对象
BigInteger i = BigInteger.valueOf(Long.MAX_VALUE);
System.out.println(i);

// 超大数运算 - 必须使用方法
i = i.multiply(BigInteger.valueOf(Long.MAX_VALUE));
System.out.println(i);  // long最大值的平方

// 更刺激的运算
i = BigInteger.valueOf(Long.MAX_VALUE);
i = i.pow(100);   // long最大值的100次方
System.out.println(i);

适用场景：需要处理超出long范围的整数计算

大数处理类 - BigDecimal

重要说明：BigDecimal也不是包装类，它是专门的数学工具类！

特点：

  • 用于精确的小数计算，解决浮点数精度问题
  • 没有对应的基本数据类型
  • 不支持自动装箱/拆箱
  • 必须使用方法进行运算
  • 不可变对象
BigDecimal i = BigDecimal.valueOf(10);
i = i.divide(BigDecimal.valueOf(3), 100, RoundingMode.CEILING);
// 计算10/3的结果，精确到小数点后100位
// RoundingMode.CEILING表示向上取整
System.out.println(i);

注意事项：

  • 对于无限循环小数，必须限制精度
  • 否则会抛出异常
  • 适用于金融计算等需要精确小数的场景

总结

包装类的作用：

  • 让基本类型支持面向对象操作
  • 提供丰富的工具方法
  • 支持自动装箱和拆箱

注意事项：

  • 理解缓存机制（-128~127）
  • 正确使用 == 和 equals() 比较
  • 合理选择 BigInteger 和 BigDecimal

应用场景：

  • 集合框架（只能存储对象）
  • 泛型编程
  • 超大数计算
  • 精确小数计算
