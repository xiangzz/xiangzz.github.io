CMD 命令行入门教程

从零开始，掌握 Windows 命令行的强大威力

第一部分：基础入门

在IDE中，点击“运行”是透明的。

命令行则让你直面这个过程。

1.1 打开与退出

打开 CMD (命令提示符)

  • 按下键盘上的 Win + R 键。
  • 在弹出的“运行”窗口中输入 cmd。
  • 按下回车键。

退出 CMD

在窗口中输入以下命令并回车：

exit

1.2 理解提示符：你的GPS

打开CMD后，你会看到类似：

C:\Users\YourName>
  • 我在哪儿？ C:\Users\YourName 是你当前的工作目录。
  • 我可以输入命令了吗？ > 符号表示系统已准备就绪。

1.3 核心导航命令 (1/3)

dir - 查看当前位置有什么

列出当前目录下的所有文件和子目录。

dir

常用参数:

  • dir /p: 分屏显示，防止内容滚动太快。
  • dir /w: 以紧凑的宽屏格式显示。

1.3 核心导航命令 (2/3)

cd - 移动到其他位置

  • 进入子目录 (例如 Desktop):
    cd Desktop
  • 返回上一级目录:
    cd ..
  • 绝对路径 vs 相对路径: 绝对路径从根目录开始，无论在哪都能准确定位。
    cd C:\Windows\System32

1.3 核心导航命令 (3/3)

切换驱动器

cd 命令不能跨盘符。要切换到D盘，直接输入盘符和冒号：

d:

要切回C盘：

c:

1.4 屏幕管理 & 1.5 获取帮助

cls - 清空屏幕

当屏幕内容杂乱时，用它来清理。

cls

获取帮助

  • help: 查看所有内置命令的列表。
  • [命令] /?: 查看特定命令的详细用法，这是你最好的朋友！
    dir /?

1.6 动手实践

  1. 打开CMD，确认当前位置 (C:\Users\YourName>)。
  2. 查看当前目录，找到 Documents 目录。
    dir
  3. 进入 Documents 目录。
    cd Documents
  4. 再次查看，对比文件资源管理器。
    dir
  5. 返回用户主目录。
    cd ..
  6. 用绝对路径进入Windows目录。
    cd C:\Windows
  7. 分页查看Windows目录。
    dir /p
  8. 清空屏幕。
    cls
  9. 预习 copy 命令。
    copy /?
  10. 退出CMD。
    exit

第二部分：文件与目录操作

本阶段的目标是让你脱离图形界面，仅用键盘就能高效地完成所有文件和文件夹的管理工作。

准备工作：创建“沙盒”

cd Desktop
mkdir cmd_practice
cd cmd_practice

2.1 创建：从无到有

mkdir (或 md) - 创建新目录

mkdir src
mkdir include

创建文件

  • 创建带内容的文件 (echo):
    echo #include <stdio.h> > src\main.c
  • 创建空文件 (type nul):
    type nul > include\utils.h

2.2 查看 & 2.3 复制

type - 显示文本文件内容

type src\main.c

copy - 复制文件

  • 创建副本:
    copy src\main.c src\main_backup.c
  • 复制到当前目录 (.):
    copy src\main.c .

2.4 移动与重命名

move - 身兼两职

  • 重命名文件:
    move main.c app.c
  • 移动文件到目录:
    move include\utils.h src\

2.5 删除：清理空间

终极警告： 命令行删除是永久性的，不会进入回收站！请再三确认！

del - 删除文件

del src\main_backup.c

rd (或 rmdir) - 删除目录

  • 删除空目录:
    rd include
  • 删除非空目录 (危险操作!):
    rd /s /q src

2.6 动手实践

一个完整的项目初始化和清理流程。

  1. 创建项目目录: mkdir MyGame
  2. 进入目录: cd MyGame
  3. 创建子目录: mkdir source, mkdir assets
  4. 创建源文件: echo int main() {} > source\game.c
  5. 创建备份: copy source\game.c game_test.c
  6. 重命名备份: move game_test.c test_backup.c
  7. 删除备份: del test_backup.c
  8. 返回上一级: cd ..
  9. 删除整个项目: rd /s /q MyGame

第三部分：程序执行与环境

理解程序是如何被操作系统发现、编译和执行的，并亲手完成从 .c 源代码到可运行程序的完整过程。

3.1 运行程序 & 3.2 环境变量 PATH

命令的本质

dir, cd, notepad 本质都是可执行程序。

notepad  // 打开记事本
calc     // 打开计算器

环境变量 PATH

PATH 是一个目录列表。系统只在这些目录里寻找你输入的命令。

查看你自己的 PATH:

path

3.3 编译并运行C代码 (1/2)

前提：安装编译器并配置PATH

  1. 安装 MinGW-w64 (GCC for Windows)。
  2. 将 MinGW 的 bin 目录 (如 C:\mingw64\bin) 添加到系统 PATH 环境变量中。
  3. 重启CMD，验证安装：
    gcc --version

3.3 编译并运行C代码 (2/2)

1. 创建源代码 hello.c

echo #include ^ > hello.c
echo int main() { >> hello.c
echo    printf("Hello from CMD!\n"); >> hello.c
echo    return 0; >> hello.c
echo } >> hello.c

注意: ^ 用于转义特殊字符 < 和 >。>> 表示追加内容到文件。

2. 编译

gcc hello.c -o hello.exe

3. 运行

因为当前目录(.)不在PATH中，需要指定路径：

.\hello.exe

3.4 命令行参数 (argc, argv)

通过命令行向C程序传递数据。

1. 创建 args.c 并编译

gcc args.c -o args.exe

2. 带着参数运行 (用空格隔开)

.\args.exe hello world 123

3. 查看输出

Total arguments: 4
argv[0]: .\args.exe
argv[1]: hello
argv[2]: world
argv[3]: 123

第四部分：提升效率的进阶技巧

学会控制命令的输入和输出，并将多个命令串联起来，形成强大的工作流。

4.1 输入/输出重定向

> (覆盖输出)

将命令结果写入文件，会覆盖原有内容。

dir > file_list.txt

>> (追加输出)

将命令结果追加到文件末尾，常用于日志记录。

echo Program started at %time% > log.txt
echo Program finished at %time% >> log.txt

(%time% 是一个代表当前时间的系统变量)

4.2 管道 (|)：命令的流水线

将前一个命令的输出，直接作为后一个命令的输入。

场景1: 查找所有 .c 文件

dir | find ".c"

场景2: 查找所有Chrome进程

tasklist 命令列出所有进程。

tasklist | find "chrome.exe"

管道让你能将简单工具组合成强大的流程。

4.3 简单的批处理脚本 (.bat)

当你需要反复执行一系列命令时，把它们写进一个 .bat 文件中。

示例: build.bat 编译并运行C程序

@echo off
echo Cleaning previous build...
del hello.exe
echo Compiling hello.c...
gcc hello.c -o hello.exe
echo Running the program:
.\hello.exe

现在，只需在CMD中运行 build 即可完成所有步骤！

4.4 动手实践

任务: 找出 C:\Windows\System32 下所有 .dll 文件，保存列表，并统计数量。

1. 保存列表

用管道连接 dir 和 find，再用重定向保存结果。

dir C:\Windows\System32 | find ".dll" > dll_list.txt

2. 统计数量

使用 find 命令的 /c (count) 参数。

dir C:\Windows\System32 | find /c ".dll"

用两行核心命令，完成了一个原本需要手动操作很久的任务。

第五部分：常用命令速查表 (1/4)

文件与目录

  • dir: 列出内容 (/p, /s)
  • cd: 切换目录 (.., \)
  • mkdir (md): 创建目录
  • copy: 复制文件
  • move: 移动或重命名
  • del: 删除文件
  • rd (rmdir): 删除目录 (/s /q 强制)
  • type: 显示文本内容
  • cls: 清屏

第五部分：常用命令速查表 (2/4)

网络相关

  • ping [host]: 测试网络连接和延迟。
  • ipconfig: 查看本机IP配置 (/all 看详情)。
  • nslookup [domain]: 查询域名对应的IP地址。
  • tracert [host]: 跟踪数据包路由路径。
  • netstat -an: 显示所有网络连接和监听端口。

第五部分：常用命令速查表 (3/4)

系统管理

  • tasklist: 显示运行中的进程。
  • taskkill /im [name] /f: 强制终止进程。
  • systeminfo: 显示详细的系统配置信息。
  • shutdown /s /t [secs]: 定时关机。
  • sfc /scannow: 扫描并修复系统文件。
  • chkdsk c: /f: 检查并修复C盘错误。

第五部分：常用命令速查表 (4/4)

学习建议

  1. 不必死记硬背：把列表当作字典来查。
  2. 多用 /?：[命令名] /? 是你最好的老师。
  3. 组合使用：用管道 | 和 find 过滤命令输出，例如：
    systeminfo | find "System Boot Time"

恭喜你，完成了本次学习！
