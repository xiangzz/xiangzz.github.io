Maven生命周期

Maven为我们提供了一套标准的项目构建流程，我们称为生命周期，它包含了很多个阶段，比如编译、测试、打包、部署等，我们只需要执行其中一个阶段的命令，那么它会自动完成该阶段之前的所有阶段，比如我们执行mvn install命令，那么它会自动完成编译、测试、打包等操作。

我们来看看Maven的三套生命周期：

  • Clean Lifecycle - 项目清理生命周期
  • Default Lifecycle - 项目部署生命周期
  • Site Lifecycle - 项目站点文档创建生命周期

我们来分别看看这三套生命周期中包含的阶段：

Clean Lifecycle

  • pre-clean - 执行一些在clean之前的工作
  • clean - 移除所有上一次构建生成的文件
  • post-clean - 执行一些在clean之后的工作

Default Lifecycle

  • validate - 验证项目是否正确，所有必要信息是否可用
  • compile - 编译项目源代码
  • test - 使用合适的单元测试框架运行测试
  • package - 将编译后的代码打包成可分发的格式，如JAR
  • verify - 对集成测试的结果进行检查，以保证质量达标
  • install - 将包安装到本地仓库，供本地其他项目使用
  • deploy - 将最终的包复制到远程仓库，供其他开发人员和项目共享

Site Lifecycle

  • pre-site - 执行一些在生成项目站点之前的工作
  • site - 生成项目的站点文档
  • post-site - 执行一些在生成项目站点之后的工作
  • site-deploy - 将生成的站点部署到特定的服务器上
